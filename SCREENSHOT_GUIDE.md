## Screenshot Guidelines

### Best Practices:
1. **Resolution**: Use at least 1920x1080 resolution for screenshots
2. **Format**: Save as PNG format for best quality
3. **Naming**: Use the suggested filenames for consistency
4. **Content**: Ensure all relevant UI elements are visible
5. **Data**: Use sample/test data that doesn't contain sensitive information
6. **Popups**: Capture popups with their parent context when possible
7. **Filters**: Show filters in their expanded state when demonstrating functionality
8. **Tables**: Include table headers and at least 2-3 sample rows

### What to Include:
- All visible UI elements in the specified section
- Relevant buttons and controls
- Form fields with labels
- Navigation elements
- Status indicators
- Error messages (if demonstrating error states)

### What to Exclude:
- Browser address bar (unless specifically needed)
- Browser bookmarks bar
- System taskbar
- Personal/sensitive data
- Unrelated browser tabs

## Notes for Screenshot Preparation

1. **Test Data**: Prepare test data before taking screenshots:
   - Sample products with images
   - Sample purchase orders
   - Sample users (staff, suppliers, members)
   - Sample transactions

2. **Browser State**: 
   - Use a clean browser window
   - Maximize browser window
   - Use consistent browser zoom level (100%)

3. **User Permissions**:
   - Ensure you're logged in with appropriate role (Staff or Supplier)
   - Have necessary permissions to view all sections

4. **Popups and Dialogs**:
   - Open popups before taking screenshots
   - Ensure popups are fully visible and not cut off
   - Show relevant form fields filled with sample data

5. **Filters and Dropdowns**:
   - Expand dropdowns to show options
   - Apply filters to show filtered results
   - Show active filter indicators
